# Feature：控制流写锁与超时

**作者**：Codex  
**日期**：2026-07-12  
**状态**：Archived

## 1. 背景（Background）

### 1.1 问题描述

`workflow_control.py` 使用原子替换防止 `tasks.md` 写出半文件，但多个进程仍可能同时读取旧内容并依次覆盖，导致先完成的状态更新丢失。

### 1.2 现状分析

- `_load()` 在加锁前读取完整 `tasks.md`。
- `_atomic_write()` 保证单次替换完整，但没有多进程互斥。
- 标准流程约定主编排器单写，但脚本本身没有失败关闭保护。

### 1.3 主要使用场景

- 多个 Codex 或 Claude Code 会话意外写入同一份 `tasks.md`。
- 上一个写进程短暂占锁，当前进程等待后继续。
- 锁长期未释放时，当前进程超时退出且不修改状态文件。

## 2. 目标（Goals）

- 所有修改 `tasks.md` 的 CLI 路径必须在同一排他锁内重新读取、决策并原子写回。
- 锁等待必须有明确超时，超时返回非 0 且不修改 `tasks.md`。
- 支持 Windows 和 POSIX，不新增第三方依赖。

### 2.1 非目标（Non-Goals）

- 不允许多个写者并行合并状态。
- 不实现分布式锁、网络文件系统一致性或锁续租。
- 不为只读的 `waves`、`dispatchable` 和 `recover` 加排他锁。

## 3. 需求细化（Requirements）

### 3.1 功能性需求

- 写操作使用与 `tasks.md` 同目录的固定锁文件。
- Windows 使用 `msvcrt.locking`，POSIX 使用 `fcntl.flock`。
- 非阻塞尝试配合单调时钟轮询，默认超时 10 秒。
- CLI 提供 `--lock-timeout`，接受大于等于 0 的秒数。
- 获锁后必须重新读取任务文件，避免锁外读取产生丢失更新。
- 成功、异常和进程退出时由操作系统释放锁。
- 超时错误包含锁文件和等待秒数。

### 3.2 非功能性需求

- Python 3 标准库，UTF-8 编码。
- 锁文件内容不作为锁状态依据，不删除正在使用的锁文件。
- 测试不得依赖真实 Agent、网络或 Git worktree。

## 4. 设计方案（Design）

### 4.1 方案概览

新增跨平台 `_task_write_lock()` 上下文管理器。CLI 识别写命令后先获取锁，再执行「读取 → 校验 → 状态转换 → 原子写回」；只读命令保持无锁。

### 4.2 组件设计（Component Design）

- 锁文件：`.<tasks.md 文件名>.lock`。
- Windows：锁定文件第 1 个字节；POSIX：对锁文件描述符加 `LOCK_EX | LOCK_NB`。
- 轮询间隔不超过 50 ms，使用 `time.monotonic()` 计算截止时间。
- 超时抛出 `TimeoutError`，由 CLI 转为退出码 2。

### 4.3 核心逻辑实现

1. 解析命令，判断是否包含 `--write`。
2. 校验 `--lock-timeout`。
3. 获取锁；失败则轮询至截止时间。
4. 获锁后调用 `_load()`，重新生成决策并写回。
5. `finally` 中解锁并关闭文件。

### 4.4 方案优劣分析

- 优点：改动集中、跨平台、进程异常退出可自动释放 OS 锁。
- 局限：仅保证本机同一文件系统上的协作进程安全；调用方绕过脚本直接写文件仍不受保护。

## 5. 备选方案（Alternatives Considered）

- `O_EXCL` 创建锁文件：实现简单，但进程崩溃会留下难以安全判定的陈旧锁，因此不采用。
- 第三方 `filelock`：接口成熟，但当前功能不值得新增依赖。

## 7. 测试计划（Test Plan）

### 7.1 单元测试

- 锁空闲时写入成功。
- 一个进程持锁时，第二个写进程超时且原文件不变。
- 持锁进程释放后，等待者成功写入。
- 非法负超时报错。

### 7.2 集成测试

- 运行全部 `agentic-framework/scripts/test_*.py`。
- 运行机器验证和 Spec Drift 检查。

### 7.3 性能测试

## 8. 可观测性与运维（Observability & Operations）

- 超时通过标准错误输出锁文件路径和超时秒数。
- 默认超时 10 秒，可通过 `--lock-timeout` 按次调整。
- 锁文件可长期存在，不表示当前仍被占用。
