# 修复 `ocr` 审查发现的问题

**状态**：Archived

## 结论

修复 `open-code-review` 对 `agentic-framework` 中 5 个 Python 脚本报告的 7 个问题，并使用 Python 标准库 `unittest` 增加回归测试，不引入第三方依赖。

## 背景

`open-code-review v1.7.7` 全文件扫描发现：规格漂移检查遗漏未跟踪文件、两个 Lint 脚本存在漏检或误报、安装脚本可能留下部分写入状态，以及 `--dry-run` 输出错误的就绪信息。

## 目标

- 修复全部 7 条已确认的问题。
- 为每类问题补充可独立运行的回归测试。
- 保持现有 CLI 参数和正常路径行为不变。

## 非目标

- 不重构无关代码。
- 不修改 `.opencodereview/`、`AGENTS.md`、`CLAUDE.md`。
- 不引入新的测试框架或运行时依赖。

## 方案

### 规格漂移检查

将已跟踪和未跟踪的规格类文件合并后参与 `_related_spec_files()` 判断，保留结果中的分类字段，避免新增 `spec.md`、`tasks.md` 或 ADR 被忽略。

### Skill 图和任务依赖 Lint

- 遍历命令文件中的全部 Skill 调用，不只检查第一个匹配。
- 区分「没有 Frontmatter」和「Frontmatter 未闭合」，对未闭合情况输出直接错误并停止从错误头部提取图关系。
- 独立判断依赖字段是否存在，不以字段值是否为空代替存在性。
- 解析任务时拒绝重复任务 ID，避免静默覆盖。

### 安装和环境配置

- 安装前完成全部冲突检查；复制失败时回滚本次已经写入的目标树，避免留下混合状态。
- `ocr` 未安装时，`--dry-run` 只输出计划动作，不输出「环境已就绪」提示；预演命令仍以成功退出表示计划生成成功。

## 权衡

- 安装回滚采用临时备份而非直接替换整个 `.codex`、`.claude` 目录，以保留框架之外的用户文件；代价是安装期间需要额外临时磁盘空间。
- 测试使用 `unittest` 而非引入 `pytest`，减少依赖，但测试样板代码会稍多。

## 验收标准

- 7 个回归场景均有自动化测试且通过。
- 5 个被审脚本通过 `python -m py_compile`。
- `python agentic-framework/scripts/lint_skill_graph.py` 退出码为 `0`；既有 orphan 警告不作为本次新增失败。
- 使用相同文件范围再次执行 `ocr scan`，不再报告本次 7 条问题。
