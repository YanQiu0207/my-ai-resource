# 前端布局骨架

## 核心规则

先排清楚，再做视觉。

必须确定：

1. 页面唯一任务。
2. 首屏主信息和主操作。
3. 页面类型。
4. 区域划分。
5. 容器模型。
6. 响应式塌缩。

## 常用骨架

| 场景 | 骨架 | 注意 |
| --- | --- | --- |
| Dashboard | 顶栏 / 侧栏 + 指标 + 图表 + 表格 | 保留信息密度，不要卡片化一切 |
| 管理后台 | App shell + 侧栏 + 主内容 + 详情区 | 主操作固定且可见 |
| 列表详情 | 左列表 + 右详情 | 移动端改为列表页 → 详情页 |
| 表单页 | 单列或分组表单 + 底部操作 | 错误贴近字段，提交有 loading |
| Landing | Hero + 证明 + 功能 + CTA | section 节奏变化，不重复模板 |
| 编辑器 | 工具栏 + 画布 + 属性面板 | 保留默认缩放、密度和面板关系 |
| 设置页 | 分类导航 + 设置项 | 危险操作独立成区 |

## 容器模型

选择一种主模型，不要混乱叠加：

- 开放留白。
- 背景色带。
- 卡片 / 面板。
- 列表 / 表格。
- 画布。
- 侧栏 / 抽屉。
- 全屏沉浸式。

如果 `ui-spec.md` 使用表格、画布、列表或色带，不要实现时改成默认卡片网格。

## Section 节奏

长页面要变化：

- 密度。
- 左右对齐。
- 图文比例。
- 背景色带。
- 媒体框比例。
- 留白。

不要整页重复「居中标题 + 三卡片」。

## 响应式

- 移动优先。
- 桌面多列，移动单列或详情页跳转。
- 侧栏在移动端变抽屉。
- 表格在移动端横向滚动或改卡片，但不能丢字段。
- 主操作在移动端仍可触达。

## 写入 ui-spec

在 `ui-spec.md` 的「布局骨架」写清：

- 页面类型。
- 区域划分。
- 主操作位置。
- 容器模型。
- 桌面布局。
- 移动端布局。
