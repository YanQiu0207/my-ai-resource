# 项目规则

## 文档写作规则（强制）

本项目所有 `.md` 文档的撰写与修改，必须遵守以下要求：

### 结构

- 先给结论，再给支撑细节，禁止先铺垫背景再引出重点
- 能用列表表达的内容不写成段落
- 不复述标题或上文已说明的信息

### 用词

- 删除套话和过渡句（如"值得注意的是""综上所述""接下来我们将"）
- 不堆砌形容词和排比句，一句话只承载必要信息
- 避免"AI 味"表达：不写空洞的总结句、不写没有信息增量的过渡段

### 边界

- 不脑补需求之外的内容，只写被要求的部分
- 不确定的事实性内容必须明确标注「不确定」，禁止用推断冒充结论

### 自检标准

写完后自问：删掉这句话，读者会不会漏掉关键信息？如果不会，删掉。

### 排版

- 中文 Markdown 排版遵循 `md-zh` 规则（中英文空格、中文与数字空格、全角标点、专有名词大小写）

## 版本管理（强制）

- 本项目使用 git 管理版本
